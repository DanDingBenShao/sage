# 字幕制作工作流

## 流程概览

```
[1] AI 识别字幕 → [2] 校对清理 → [3] 时间轴精调
     → [4] 样式设计 → [5] 导出(烧录/SRT)
```

## 工具选择

| 场景 | 推荐 | 为什么 |
|------|------|--------|
| 短视频快速出片 | **剪映/CapCut** | AI 识别最准，免费，一站式 |
| 长视频精细打轴 | **Arctime** | 波形图拖拽打轴，多语言轨道 |
| 纯文本批量处理 | **Subtitle Edit** | 开源免费，波形图+批量操作 |
| 特效字幕 (.ass) | **Aegisub** | 高级样式、动画、karoake 效果 |

## 推荐混合流程

```
剪映 AI 识别 (快速初稿)
  → 导出 SRT
  → Arctime 精调时间轴 + 样式
  → 导出最终字幕文件
```

## Step 1: AI 识别

```
剪映/CapCut 操作:
  文本 → 智能字幕 → 开始识别
  - 选对源语言 (中文选"中文")
  - 可选: 双语字幕 (中+英翻译)
  - 可选: 关键词高亮
```

## Step 2: 校对清单

```
[ ] 专有名词正确 (人名/品牌/产品名)
[ ] 数字格式统一 ("10" vs "十")
[ ] 标点补充 (逗号/句号断句)
[ ] 删除填充词 ("嗯""那个""就是说")
[ ] 合并碎片 (<1秒的短字幕)
[ ] 拆分长句 (>6秒的拆到自然断点)
```

## Step 3: 时间轴规范

| 规范 | 值 | 说明 |
|------|-----|------|
| 最短时长 | ≥ 1 秒 | 否则闪一下就没了 |
| 最长时长 | ≤ 6-7 秒 | 否则观众读完前就切了 |
| 中文每行 | ≤ 20 字 | 短视频可更短 (12-15字) |
| 英文每行 | ≤ 42 字符 | 标准字幕规范 |
| 提前量 | 出现比语音早 0.1-0.2s | 阅读反应时间 |
| 延后量 | 消失比语音晚 0.1-0.3s | 看完再切 |

## Step 4: 样式设计

```
字体: 无衬线 (黑体/思源黑体/Roboto)
      不要用衬线体/花体/手写体 → 密集对话时辨识度差

字号: 1080p 视频，中文字体 40-56px，英文 32-48px
      优先在手机上测试可读性

颜色方案 (三选一):
  A. 白字 + 黑色描边 (2-4px) + 轻微阴影 ← 最通用
  B. 白字 + 半透明黑底框 ← 画面极亮/极杂乱时用
  C. 黄字 + 黑色描边 ← 传统电影风格

对比度: ≥ 4.5:1 (WCAG AA 级)
  不要: 浅色字无描边 / 灰色字 / 白字+浅色底框

位置: 底部居中，上方留安全边距
      避开平台 UI (TikTok 右侧按钮/YouTube 进度条)

动效: 不要弹跳/旋转/逐字弹入
      可用: 关键词逐字高亮 (教程类强调操作)
```

## Step 5: 导出

```
短视频平台 (TikTok/抖音/Reels):
  → 烧录进画面 (MP4 自带字幕)
  → 大字 + 居中 + KTV 跟读高亮风格

长视频平台 (YouTube/B站):
  → SRT 文件 (独立上传，观众可开关)
  → 底部标准字幕样式
```

## 导出前自查

- [ ] 静音看一遍 → 所有信息能理解
- [ ] 每条约 1-7 秒
- [ ] 断句在自然语义停顿处
- [ ] 字体统一，有描边，对比度够
- [ ] SDH 元素补充 ([音乐]、[掌声]、说话人标注)
