# 脚本与分镜方法论

## 核心公式: Hook → Story → Payoff

```
Hook (5-15s)   → 抓住注意力
  ├── 提出问题 / 展示结果 / 制造悬念 / 打破常识
Story (主体)    → 展开内容
  ├── 教程: Step 1 → Step 2 → Step 3 → 结果展示
  ├── 科普: 问题 → 原因 → 方案 → 总结
  └── Vlog: 起点 → 过程 → 转折 → 终点
Payoff (结尾)   → 给观众一个交代
  ├── 总结要点 / 行动号召 / 下期预告 / 情感收尾
```

## 钩子类型与模板

| 类型 | 模板 | 适用 |
|------|------|------|
| **问题式** | "你知道为什么 [常见问题] 吗？其实..." | 教程、科普 |
| **结果前置** | "用这个方法，我 [具体成果]" | 教程、工具推荐 |
| **反常识** | "[常识] 其实是错的。真相是..." | 科普、观点 |
| **故事开场** | "上周我遇到了一个 [具体情境]..." | Vlog、经验分享 |
| **数字冲击** | "[XX%] 的人不知道 [事实]" | 科普、数据 |

## 脚本字数-时长换算

```
中文语速参考:
  正常讲解: 每分钟 180-220 字
  教程演示: 每分钟 150-180 字 (有操作停顿)
  快节奏口播: 每分钟 250-300 字

估算公式:
  目标时长(分钟) × 200 字/分钟 = 脚本目标字数
  
  例: 8 分钟教程 ≈ 1600 字脚本
```

## 教程类脚本模板

```
[0:00-0:15] 钩子
  一句话说清楚这期视频教什么 + 学完能干什么

[0:15-0:45] 前置说明
  需要什么工具/环境/基础 → 让不对口的观众提前离开

[0:45-7:00] 主体 (按步骤拆)
  每个步骤:
    1. 操作演示 (屏幕录制/实拍)
    2. 讲解为什么这样做
    3. 展示中间结果
  步骤间加过渡句: "接下来..."

[7:00-8:00] 总结 + CTA
  回顾 3 个关键点 + 引导评论/点赞/下期预告
```

## 科普类脚本模板

```
[0:00-0:15] 钩子
  提出一个问题或反常识的事实

[0:15-1:00] 问题展开
  这个问题为什么重要？影响谁？

[1:00-7:00] 分析和解答 (2-3 个角度)
  每个角度: 事实 → 解释 → 例子/类比

[7:00-8:00] 总结观点 + CTA
```

## 分镜表模板

| 时间 | 画面 | 音频 | 素材来源 | 备注 |
|------|------|------|---------|------|
| 0:00-0:15 | 主持人出镜 / 标题卡 | "今天教你..." | 实拍 | 钩子 |
| 0:15-0:30 | 屏幕录制: 打开工具 | "首先..." | 录屏 | |
| 0:30-1:00 | 截图标注 #1 + B-roll | "这里注意..." | 截图+Stock | |
| 1:00-2:00 | 屏幕录制: 操作步骤 | 讲解 | 录屏 | |
| ... | ... | ... | ... | ... |
| 7:00-8:00 | 要点文字卡 + 结尾 | 总结 + CTA | 文字卡片 | |

## 分镜设计原则

1. **每 15-30 秒换一次画面** — 维持视觉新鲜感
2. **关键步骤配截图标注** — 不要只靠旁白描述
3. **B-roll 服务于内容** — 提到什么就展示什么画面
4. **情绪段落用留白** — 不要一直塞满画面和声音
5. **画图优于说话** — 复杂概念画图比口述效率高 3 倍

## 脚本自检清单

```
[ ] 钩子在前 3 秒说了什么？能不能让目标观众停下来？
[ ] 一句话能概括这期视频吗？说不出来就是主题不聚焦
[ ] 有没有"这步为什么要做"的解释？不能只有操作没有原因
[ ] 有没有"废话段落"？读一遍脚本，每段问"删掉它观众会 miss 什么？"
[ ] 结尾有没有告诉观众下一步做什么？
[ ] 脚本读完用时是否在目标的 ±20% 以内？
```
